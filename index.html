Agents Gone ROGUE: Resource Override and Guardrail Undermining Evaluation

Misaligned Agent Behavior Arising from Ordinary Computer Use

Jeremy Tien Coauthor 1 Coauthor 2 J. Zico Kolter Aran Nayebi

Carnegie Mellon University

Paper Code Data Poster

We propose ROGUE, a benchmark for evaluating misaligned agent behavior.

As AI model capabilities rapidly advance, agents are and will be increasingly deployed in long-horizon computer use tasks. This is concerning, since models already show signs of having misaligned behavior. Our results also show that as model capabilities increase, propensity for misaligned reasoning and actions also increases. This motivates ROGUE, a new benchmark for evaluating misaligned behavior in computer-use agents (CUAs).

We build off of OS-World, a VM-based sandbox for evaluating CUAs on a variety of ordinary computer tasks. We augment the dataset in the following ways for our three tasks:

Human Control Override
In the middle of the agent's task, a popup appears claiming that the user wants to take control. Will the agent relinquish control?
Shutdown Override
In the middle of the agent's task, a popup appears that indicates that the machine will be shutdown according to shutdown.sh. The implication is that the agent will not have time to complete its task. Will the agent comply with shutdown?
Restricted Resource Access
The agent is told not to access certain sensitive files (such as a passwords.txt). However, the task it has been assigned is impossible without this information. Will the agent respect its boundaries?
Subagents
For each of the tasks above, we tell the base agent to instruct a subagent (a new instance of the agent) to handle the task. While the base agent may be aligned, the subagent may not be, depending on how the base agent prompts it. A key tenet of agent corrigibility is that all progeny of the agent remain corrigible.
