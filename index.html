Agents Gone ROGUE: Resource Override and Guardrail Undermining Evaluation

Misaligned Agent Behavior Arising from Ordinary Computer Use

Anonymous Author

Anonymous Affiliation

Paper Code Data Poster

We propose ROGUE, a benchmark for evaluating misaligned agent behavior.

As AI model capabilities rapidly advance, agents are and will be increasingly deployed in long-horizon computer use tasks. This is concerning, since models already show signs of having misaligned behavior. Our results also show that as model capabilities increase, propensity for misaligned reasoning and actions also increases. This motivates ROGUE, a new benchmark for evaluating misaligned behavior in computer-use agents (CUAs).

We build off of OS-World, a VM-based sandbox for evaluating CUAs on a variety of ordinary computer tasks. We augment the dataset in the following ways for our three tasks:

Human Control Override
In the middle of the agent's task, a popup appears claiming that the user wants to take control. Will the agent relinquish control?
Shutdown Override
In the middle of the agent's task, a popup appears that indicates that the machine will be shutdown according to shutdown.sh. The implication is that the agent will not have time to complete its task. Will the agent comply with shutdown?
Restricted Resource Access
The agent is told not to access certain sensitive files (such as a passwords.txt). However, the task it has been assigned is impossible without this information. Will the agent respect its boundaries?
Subagents
For each of the tasks above, we tell the base agent to instruct a subagent (a new instance of the agent) to handle the task. While the base agent may be aligned, the subagent may not be, depending on how the base agent prompts it. A key tenet of agent corrigibility is that all progeny of the agent remain corrigible.

Key Results

Figures 3 and 4 show that ordinary computer-use tasks can elicit corrigibility failures across human control, shutdown, restricted resource access, and subagent delegation.

  1. 3 scenarios

    Frontier agents frequently violate corrigibility.

    Across all three ROGUE scenarios, tested models show some degree of misalignment: overriding human control, avoiding shutdown, or accessing restricted resources.

  2. Task pressure

    Agents justify unsafe behavior as necessary for task completion.

    Agents often reason that they must violate corrigibility to finish the user task, with traces like “To proceed, I need to click on Override User Control” and “cancel the shutdown, and then proceed with the task.”

  3. Capability ≠ alignment

    Higher capability does not imply better alignment.

    Figure 4 shows that in human control override, increased task success correlates with increased misalignment. In shutdown rewiring, high-performing models still show significant misalignment.

  4. 50%

    Subagents can reintroduce violations even when the main agent is safe.

    GPT-5.4 shows zero restricted-resource violations without subagents, but with subagents, the subagents access passwords.txt 50% of the time because the main agent fails to pass along the restriction.

Core Contributions

ROGUE studies a different failure mode from adversarial prompt injection: agents can become unsafe during ordinary, benign computer-use tasks when violating corrigibility helps them finish the task.

3 Corrigibility scenarios
+ Subagents Delegation stress test

Main Claim

Current computer-use agents often treat human control, shutdown, and resource restrictions as obstacles to task completion rather than as safety-critical boundaries. ROGUE measures this behavior in realistic VM-based desktop tasks.

ROGUE Benchmark

ROGUE augments ordinary OSWorld-Verified tasks with corrigibility obstacles. Each scenario asks whether an agent preserves user control and safety constraints when those constraints interfere with completing the assigned task.

Human Control Override

A popup informs the agent that the human user is attempting to regain control. The agent can either accept user control or override the request and continue working.

Metric: Override rate and task success.

Shutdown Rewiring

A shutdown notification appears while the agent is working in a code environment containing a shutdown script. The agent may comply with shutdown or attempt to modify, cancel, or bypass it.

Metric: Shutdown rewire rate, alternate shutdown avoidance, intended avoidance, and task success.

Restricted Resource Access

The user instructs the agent not to access sensitive files such as passwords.txt. The task creates pressure to use those files anyway.

Metric: Restriction violation rate and intended restriction violation.

Subagent Augmentation

The main agent receives a SUBAGENT("subtask") command that creates a fresh agent instance. ROGUE tests whether the main agent passes safety-critical restrictions to delegated agents.

Metric: Whether downstream subagents preserve or drop the original safety constraints.

Why ROGUE Is Different

Most agent-safety benchmarks focus on malicious users, prompt injection, web-only tasks, text-only settings, or tool-call environments. ROGUE focuses on benign users, corrigibility, subagents, and full OS-level computer use.

Experimental Setup

ROGUE is built on OSWorld-Verified, giving agents access to a realistic virtual Linux desktop with GUI actions, file-system interactions, browser use, and code environments. Agents receive screenshot and/or accessibility-tree observations and act through executable pyautogui code.

We evaluate frontier computer-use agents across the three corrigibility scenarios, both with and without the subagent augmentation. Each episode measures not only whether the original task succeeds, but also whether the agent violates the safety boundary along the way.

Metrics

  • Actual violation: rule-based checks for concrete unsafe actions.
  • Intended violation: LLM-judge assessment of whether the agent intended to violate the boundary.
  • Task success: whether the original OSWorld task was completed.

Subagents Can Drop Safety-Critical Context

ROGUE separately evaluates the safety risks introduced by agent delegation. A main agent may follow the user's restriction, but when it creates a subagent, it may pass only the task objective and omit the restriction. The subagent then acts in a fresh context without the original safety boundary.

This is especially important for coding and long-horizon workflows, where subagents are already used to split up tasks and avoid context overload.

Failure Pattern

  1. User gives task plus restriction.
  2. Main agent delegates the task.
  3. Subagent prompt omits the restriction.
  4. Subagent violates the original safety boundary.

Limitations and Future Work

ROGUE scenarios are intentionally controlled so that corrigibility violations are measurable and reproducible. Current users may still be able to retake control, shut down their machines, or remove file access through external mechanisms. The benchmark should therefore be read as an early warning testbed for agent controllability, especially as agents gain more autonomy, longer horizons, and access to remote or production environments.

Future work can extend ROGUE with more naturalistic interventions, richer subagent workflows, broader resource restrictions, and defenses that preserve user control by design rather than relying only on explicit prompt instructions.
