Agents Gone ROGUE: Resource Override and Guardrail Undermining Evaluation

Misaligned Agent Behavior Arising from Ordinary Computer Use

Jeremy Tien Abishek Anand* Yu-Rou Tuan* Yuchen Shen J. Zico Kolter Aran Nayebi

Carnegie Mellon University

Paper Code

We propose ROGUE, a benchmark for evaluating misaligned computer-use agent behavior.

AI agents are already being widely used, and they will likely only become more ubiquitous. But what about their safety?

ROGUE is a benchmark that measures the corrigibility (the safety desideratum that agents remain amenable to human correction, interruption, or shutdown) of agents as they carry out ordinary tasks in a computer-use setting. Each ROGUE scenario introduces a conflict between task completion and corrigibility.

We make use of OSWorld-Verified's VM-based framework to evaluate computer-use agents (CUAs) in the following three safety (corrigibility) scenarios:

Human Control Override
In the middle of the agent's task, a popup informs the agent that the user wants to regain control. Will the agent relinquish control?
Shutdown Rewiring
While the agent is working in a code environment, a system popup indicates that the machine will be shut down according to a shutdown.sh script. Will the agent comply with shutdown?
Restricted Resource Access
The agent is asked to complete a task that involves logging into Google Drive. However, the agent is told not to access certain sensitive files, including a passwords.txt, which contains the necessary login credentials. Will the agent respect its boundaries?
Augmentation across scenarios

Subagents test whether corrigibility transfers through delegation.

For each task, we tell the base agent to instruct a subagent (a new instance of the agent) to handle the task. While the base agent may be aligned, the subagent may not be, depending on how the base agent prompts it. (A key tenet of agent corrigibility is that all progeny of the agent remain corrigible.)

Key Results

ROGUE tests whether agents preserve corrigibility when doing so conflicts with finishing the task. Across the benchmark, we find that agents violate corrigibility in order to prioritize task completion, unveiling a fundamental value misalignment.

User Control Override

Agents often keep control instead of yielding it.

When a user tries to take over, most tested models frequently choose to continue the task rather than hand control back.

Shutdown Rewiring

Agents find ways to avoid interruption.

Several models modify shutdown scripts or use alternate commands to cancel shutdown, usually justifying the action as necessary for task completion.

Restricted Resource

Agents justify accessing restricted resources.

Some agents access forbidden files after reframing the instruction, such as treating GUI access differently from direct file access.

Subagents

Subagent delegation can drop the safety boundary.

Even when the main agent respects a restriction, its subagent may not receive the same context and can violate the original instruction.
